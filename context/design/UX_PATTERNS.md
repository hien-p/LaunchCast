# UX Patterns

> Generated by setup wizard from PRD.

## Navigation

- **Single-page app** with tab-style navigation
- Primary screens: Vibe Setup → Generation → Player → (Live Mode) → (Trend Radar)
- No traditional nav bar — flow is linear for first-time use
- Returning users land on the player with today's episode

## Key Flows

### First Visit
1. Vibe Setup (select interests, meet hosts, preview voice)
2. CTA: "Start Listening to Today's Hunt"
3. Live Generation screen (watch episode being built)
4. Auto-transition to Player when complete

### Return Visit
1. Land on Player with today's episode
2. Browse past episodes in archive
3. Access Vibe Setup from settings to change preferences

## Loading States

- **Generation in progress:** Full-screen progress view with:
  - Step indicator (Scraping → Analyzing → Writing → Recording)
  - Live URL display showing which site Firecrawl is scraping
  - Product cards that pop in as each product is scraped
  - Timing stats and credit usage
- **Audio loading:** Waveform shimmer animation in player
- **Deep dive opening:** Skeleton cards for pricing/features sections

## Empty States

- **No episodes yet:** "Hit 'Generate' to create your first episode — it takes about 3 minutes."
- **No past episodes:** "This is your first episode! Come back tomorrow for more."

## Error Handling

- **Scraping fails for a product:** Card shows with "Couldn't reach this site" badge; podcast script acknowledges it naturally
- **Generation fails entirely:** Clear error message with "Try Again" button; no raw error codes
- **Audio won't play:** Fallback to download link

## Player Patterns

- Sticky bottom bar on mobile with mini player
- Tap product in show notes → seek to that segment
- Segmented progress bar colored by product
- Per-host ratings (Aero orange / Nova purple) displayed inline

## Accessibility

- All interactive elements keyboard navigable
- Audio player has visible controls (no autoplay without user action after first visit)
- Color is not the only differentiator — Aero/Nova also distinguished by labels and icons
- Sufficient contrast ratios on dark backgrounds
- Screen reader labels for player controls
