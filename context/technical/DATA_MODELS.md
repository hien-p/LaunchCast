# Data Models

> Generated by setup wizard from PRD.

## Entities

### Episode
```json
{
  "id": "string (date format: YYYY-MM-DD)",
  "date": "string (YYYY-MM-DD)",
  "title": "string",
  "duration_seconds": "integer",
  "audio_url": "string (relative path to MP3)",
  "products": "Product[]",
  "script": "ScriptLine[]",
  "created_at": "string (ISO 8601 datetime)"
}
```

### Product
```json
{
  "name": "string",
  "tagline": "string",
  "ph_url": "string (URL)",
  "website_url": "string (URL)",
  "upvotes": "integer",
  "ph_description": "string",
  "website_content": "string (markdown from Firecrawl)",
  "pricing": {
    "free_tier": "boolean",
    "plans": "array of plan objects"
  },
  "features": "string[]",
  "aero_rating": "integer (1-10)",
  "nova_rating": "integer (1-10)",

  "segment_end_seconds": "integer"
}
```

### ScriptLine
```json
{
  "speaker": "string (AERO | NOVA)",
  "text": "string",
  "product_ref": "string | null",
  "type": "string (intro | analysis | hot_take | rating | transition | outro)"
}
```

### UserPreferences (localStorage only — not stored server-side)
```json
{
  "interests": "string[] (e.g. ['Open Source', 'LLMs', 'Frontend Tools'])",
  "deep_dives_only": "boolean",
  "technical_detail": "boolean"
}
```

## Relationships

- An **Episode** contains many **Products** (embedded, not a separate table)
- An **Episode** contains many **ScriptLines** (embedded, ordered array)
- A **ScriptLine** references a **Product** via `product_ref` (by name)
- **Products** within an episode have segment timestamps linking audio position to product

## Conventions

- Episode IDs are date-based (one episode per day)
- All timestamps are ISO 8601 UTC
- Product data is denormalized within episodes (no separate product table)
- SQLite used for episode index; full episode data also stored as JSON files for easy access
- No soft deletes — episodes are append-only
