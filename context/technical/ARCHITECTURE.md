## System Overview

LaunchCast is a pipeline system with four stages and a delivery layer:

```
Scheduler → Scraping Pipeline → Script Generator → Voice Synthesis → Delivery Layer
```

- **Scraping Pipeline** (Firecrawl): Scrapes PH top launches, then deep-scrapes each product website
- **Script Generator** (Claude API): Synthesizes scraped data into a two-host podcast conversation
- **Voice Synthesis** (ElevenLabs + FFmpeg): Converts script to audio, stitches into single MP3
- **Delivery Layer** (React frontend + FastAPI backend): Web player, SSE progress, deep dive modals

The backend orchestrates the pipeline and serves the frontend via REST + SSE.
The frontend is a standalone SPA that consumes the backend API.

## Data Flow

### Episode Generation (Primary Flow)
1. Trigger (manual or cron) → FastAPI endpoint
2. Firecrawl Search API scrapes PH top launches → structured product list
3. For each product: Firecrawl scrapes the product website → markdown content
4. All product data sent to Claude API → structured podcast script (JSON array of speaker/text lines)
5. Script lines routed to ElevenLabs TTS by speaker → individual audio clips
6. FFmpeg stitches clips in order with pauses → single MP3 episode
7. Episode metadata + audio saved to SQLite + filesystem
8. Frontend notified via SSE throughout steps 2-6

### Live Listening (Secondary Flow)
1. User opens web player → fetches latest episode metadata from API
2. Audio streams from backend file server
3. Show notes and product links rendered alongside player

### Live Interactive Mode (P1)
1. User speaks → ElevenAgents ASR
2. LLM processes with episode context → may call Firecrawl Server Tool for live search
3. Response synthesized via ElevenAgents TTS → voice response
4. Client tool updates UI with sources/links

## Key Design Decisions

- **SSE over WebSockets:** Simpler for one-way server-to-client progress updates during generation
- **SQLite over Postgres:** Zero-config for hackathon; single-file database travels with the project
- **Separate frontend/backend:** Allows independent deployment (Vercel + Railway) and clear API boundary
- **Sequential audio generation:** ElevenLabs clips generated one at a time to avoid rate limits, then stitched
- **JSON files as intermediate storage:** Product data and scripts saved as JSON for easy debugging and caching
