# Tech Stack

> Generated by setup wizard from PRD. Update as decisions evolve.

## Frontend
- **Framework:** React 18+ with Vite
- **Styling:** Tailwind CSS
- **Audio:** ElevenLabs React SDK for voice playback and live interactive mode
- **Real-time:** EventSource API for SSE (live generation progress)
- **State:** React built-in state (useState/useContext) — no heavy state library needed

## Backend
- **Runtime:** Node.js 20+ / TypeScript
- **Framework:** Express.js with TypeScript (lightweight, SSE streaming support)
- **API Style:** REST + Server-Sent Events (SSE) for real-time progress

## Database
- **Engine:** SQLite (via better-sqlite3)
- **Storage:** JSON files for episode metadata, SQLite for structured queries
- **Rationale:** Minimal infrastructure for hackathon — no external DB service needed

## Auth
- None in v1 — no user accounts
- Preferences stored in localStorage on the client

## Infrastructure
- **Frontend hosting:** Vercel (free tier, instant deploy)
- **Backend hosting:** Railway or Render (free tier, Node.js support)
- **Audio storage:** Local filesystem or object storage (episodes as MP3 files)
- **Scheduler:** Cron (local) or cloud scheduler for daily 6 AM generation

## Key Third-Party Services

| Service | Purpose | Tier |
|---------|---------|------|
| **Firecrawl** | Web scraping (PH + product sites) | ~22 credits/day |
| **Claude API (Sonnet)** | Podcast script generation | ~$0.05/episode |
| **ElevenLabs** | Text-to-speech (two voices) | Free/Starter tier |
| **FFmpeg** | Audio stitching + processing | Open source (local) |

## Rationale
- **React + Vite over Next.js:** Lightweight SPA is sufficient — no SSR needed, and ElevenLabs React SDK has first-class Vite support
- **TypeScript everywhere:** Same language for frontend and backend — shared types, faster iteration, single toolchain
- **Express over FastAPI:** Lightweight, familiar, excellent SSE support. Firecrawl and ElevenLabs both have JS/TS SDKs
- **SQLite over Postgres:** Zero-config, single-file database — perfect for hackathon pace. Can migrate to Postgres later if needed
- **No auth:** Listeners don't need accounts — the app generates and serves public podcast content
