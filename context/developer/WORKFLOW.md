# Development Workflow

> Generated by setup wizard. Solo developer + Claude Code.

## Branch Strategy

- **main:** Always deployable. All work lands here.
- **Feature branches:** `feat/[feature-name]` for larger features (optional for hackathon pace)
- During rapid hackathon iteration, committing directly to main is acceptable.

## Commit Style

Conventional commits, short and clear:

```
feat: add PH scraping pipeline with Firecrawl
fix: handle missing pricing data in product scrape
chore: add ElevenLabs SDK dependency
```

Prefixes: `feat`, `fix`, `chore`, `refactor`, `docs`, `test`

## PR / Review Process

- Solo project — no PRs required during hackathon
- Use `code-reviewer` agent before committing significant features
- Post-hackathon: feature branch → PR → review → merge

## Local Development

### Backend
```
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env  # fill in API keys
uvicorn app.main:app --reload --port 8000
```

### Frontend
```
cd frontend
npm install
cp .env.example .env  # set VITE_API_URL
npm run dev
```

### Both (typical dev session)
- Terminal 1: Backend on port 8000
- Terminal 2: Frontend on port 5173 (Vite default)
- Frontend proxies API calls to backend

## Testing

- Run backend tests: `pytest` from `backend/`
- Run frontend tests: `npm test` from `frontend/`
- Use `test-writer` agent after completing features
