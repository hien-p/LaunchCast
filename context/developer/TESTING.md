# Testing Strategy

> Generated by setup wizard. Expand as test coverage grows.

## Philosophy

Test the pipeline stages independently. Each stage (scraping, script gen, TTS, stitching) should be testable with mock data so development doesn't burn API credits.

## Unit Tests

### Backend (pytest)
- Scraper data parsing (given mock HTML/markdown, extract correct product data)
- Script generator output format (given mock product data, validate script structure)
- Audio stitcher logic (given mock clip metadata, validate correct ordering and pause insertion)

### Frontend (Vitest)
- Component rendering (player controls, product cards, progress indicators)
- SSE event parsing
- Preference storage/retrieval from localStorage

## Integration Tests

- Full pipeline with mock Firecrawl responses (no real API calls)
- API endpoint response format validation
- SSE event stream format validation

## E2E Tests

- Deferred to post-hackathon
- Key paths: Vibe Setup → Generate → Listen flow

## Coverage Goals

- Hackathon: Focus on pipeline correctness, not coverage %
- Post-hackathon: 80% backend, 60% frontend
